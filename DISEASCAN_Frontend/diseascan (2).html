DiseaScan is a student-built diagnostic support tool that uses computer vision to help identify common skin conditions from uploaded images.

What We're Building

DiseaScan is designed to support — not replace — clinical judgment. The goal is to give clinicians and patients a fast second opinion on visible skin conditions, backed by a model trained on publicly available dermatology datasets. We're in active development and improving the model continuously.

How the
model works.

A walkthrough of the architecture, training decisions, and inference pipeline that powers DiseaScan.

layers

Base Architecture

The model uses EfficientNetB4 as its backbone, loaded with ImageNet weights and fine-tuned on a dermatology dataset. EfficientNet was chosen for its strong performance-to-parameter ratio — it achieves competitive accuracy without the memory overhead of larger networks. The input resolution is fixed at 380×380 pixels, which matches EfficientNetB4's optimal input size.

Training Setup

  • › Transfer learning from ImageNet pretrained weights
  • › Custom WarmupCosineDecay learning rate schedule
  • › Linear warmup phase to stabilise early training
  • › Cosine decay for smooth convergence
target

Loss Function

Uses a custom FocalLoss implementation (γ=2.0) to handle class imbalance — the dataset has significantly more benign cases than malignant ones.

  • › Label smoothing (ε=0.10) to reduce overconfidence
  • › Focal weighting upsamples hard examples
analytics

Stable Inference Pipeline

Rather than running a single forward pass, DiseaScan runs 3–5 passes and averages the softmax probability distributions. This reduces sensitivity to minor preprocessing variations and produces more stable confidence scores.

1

Validate & decode image

Magic byte check, PIL decode, corruption detection

2

Resize to 380×380, EfficientNet normalisation

Pixel values scaled to [−1, 1] to match training preprocessing

3

N forward passes, average softmax outputs

Runs in a thread pool to avoid blocking the FastAPI event loop

4

Re-normalise probabilities, extract top-3

Floating-point drift corrected, ranked predictions returned

5

Enrich with clinical metadata

Full name, severity, description, and recommended action attached to each prediction

warning

Known Limitations

  • › Model is trained on dermoscopy images — performance on plain photographs may be lower
  • › Class imbalance in the training dataset means rare conditions have less training signal
  • › CPU-only inference on this setup — each request takes 1–5 seconds depending on run count
  • › No Grad-CAM heatmap yet — explainability layer is planned but not implemented

Ethics &
Privacy.

What we do with your data, how decisions are made, and where our model's responsibility ends.

shield_lock

Data Handling

  • Images deleted after inference

    Uploaded images are processed in memory and never written to disk or stored on the server.

  • No third-party data sharing

    We do not send image data to any external API, analytics service, or third party.

  • No retraining without consent

    Uploaded images are never used to retrain the model. Any future feedback loop would require explicit opt-in.

gavel

Compliance

  • HIPAA-aware design

    The system is designed with HIPAA principles in mind. We are not yet formally certified for clinical deployment.

  • Medical disclaimer on every response

    Every API response includes a disclaimer. The model is a research tool, not a diagnostic device.

  • Auditable predictions

    Every inference is logged with its input hash, prediction, confidence, and latency for review and auditing.

Where Our Responsibility Ends

DiseaScan returns a probability distribution, not a diagnosis. The difference matters. A high confidence score for melanoma is a signal to seek evaluation — it is not a confirmed finding. We designed the output format to communicate this: severity labels are accompanied by recommended actions, and every result ends with a medical disclaimer.

We also surface uncertainty explicitly. When the model is unsure — when the top prediction has low confidence or the top-3 distribution is flat — the confidence label will say "Low" and the stability note will reflect it. We do not round up confidence scores or suppress uncertain results.
